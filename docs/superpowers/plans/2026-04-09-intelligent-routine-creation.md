# Intelligent Routine Creation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers-extended-cc:subagent-driven-development (recommended) or superpowers-extended-cc:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship intelligent Routine creation with two paths (DM one-shot + Routine UI multi-turn) plus an onboarding integration that turns the user's selected tasks into draft routines at workspace provisioning time.

**Architecture:** Introduce a `draft` status on `routine__routines`, plus three tracking columns (`creation_channel_id`, `creation_session_id`, `source_ref`). A new claw-hive component (`team9-routine-creation`) provides the `createRoutine`/`getRoutine`/`updateRoutine` tool suite. The DM path reuses the user's normal agent with the component in "DM mode" (`createRoutine` tool only). The Routine UI path registers a short-lived **per-routine agent clone** whose `extraComponentConfigs['team9-routine-creation']` has `routineId` baked in — this avoids the per-session `team9Context` mechanism and keeps the clone cleanly scoped to one routine. The onboarding provisioning pipeline gains a new step that converts selected task titles into draft routines with `source_ref = onboarding:{recordId}:{taskId}` for idempotency.

**Tech Stack:** NestJS 11 + Drizzle ORM + PostgreSQL on the backend; @team9claw/claw-hive component system in `team9-agent-pi/packages/claw-hive`; React 19 + TanStack Query + TanStack Router on the client; Jest (server) + Vitest (claw-hive) for tests.

**Design Doc:** [docs/superpowers/specs/2026-04-09-intelligent-routine-creation-design.md](../specs/2026-04-09-intelligent-routine-creation-design.md) (Revision 3).

**Depends on:** Bug-fix commits `76294882` (team9) and `933fe8d` (team9-agent-pi) that already landed the per-event `team9Context` plumbing for `team9-staff-bootstrap`. This plan does **not** use that mechanism for routine creation — it uses per-routine agent clones with static config — but the bug fix must be deployed alongside Phase 1 to avoid regressing bootstrap.

```typescript
  /**
   * System helper to archive a routine creation channel.
   *
   * Unlike {@link archiveChannel}, this method:
   * - Does NOT enforce owner/admin role (system-initiated during routine
   *   complete-creation).
   * - ACCEPTS direct channels, because creation channels are DMs between
   *   the user and the clone agent's shadow user.
   * - Is idempotent: no-op if channel is missing or already archived.
   *
   * Callers: {@link RoutinesService.completeCreation} and
   * {@link RoutinesService.delete} (when deleting a draft).
   */
  async archiveCreationChannel(
    channelId: string,
    tenantId?: string,
  ): Promise<void> {
    const conditions = [eq(schema.channels.id, channelId)];
    if (tenantId) {
      conditions.push(eq(schema.channels.tenantId, tenantId));
    }

    const [channel] = await this.db
      .select({
        id: schema.channels.id,
        type: schema.channels.type,
        isArchived: schema.channels.isArchived,
      })
      .from(schema.channels)
      .where(and(...conditions))
      .limit(1);

    if (!channel) {
      this.logger.debug(
        `archiveCreationChannel: channel ${channelId} not found, skipping`,
      );
      return;
    }
    if (channel.isArchived) {
      this.logger.debug(
        `archiveCreationChannel: channel ${channelId} already archived, skipping`,
      );
      return;
    }

    await this.db
      .update(schema.channels)
      .set({
        isArchived: true,
        updatedAt: new Date(),
      })
      .where(and(...conditions));

    await this.redis.invalidate(REDIS_KEYS.CHANNEL_CACHE(channelId));
  }
```

```typescript
async update(
  routineId: string,
  dto: UpdateRoutineDto,
  userId: string,
  tenantId: string,
): Promise<schema.Routine> {
  const routine = await this.getRoutineOrThrow(routineId, tenantId);
  this.assertCreatorOwnership(routine, userId);

  // Draft → upcoming transitions MUST go through completeCreation so that
  // required-field validation, creation channel archiving, and clone agent
  // cleanup all run together. Reject here to prevent accidental shortcut.
  if (dto.status !== undefined && dto.status !== routine.status) {
    throw new BadRequestException(
      'Cannot change routine status via update. Use POST /v1/routines/:id/complete-creation to finalize a draft, or the dedicated start/pause/resume/stop endpoints for runtime transitions.',
    );
  }

  // 1. Write the scalar routines-table columns
  const updateData: Record<string, unknown> = { updatedAt: new Date() };
  if (dto.title !== undefined) updateData.title = dto.title;
  if (dto.botId !== undefined) updateData.botId = dto.botId;
  if (dto.description !== undefined) updateData.description = dto.description;
  if (dto.scheduleType !== undefined) updateData.scheduleType = dto.scheduleType;
  if (dto.scheduleConfig !== undefined)
    updateData.scheduleConfig = dto.scheduleConfig;

  const [updated] = await this.db
    .update(schema.routines)
    .set(updateData)
    .where(eq(schema.routines.id, routineId))
    .returning();

  // 2. If documentContent was provided, update the linked document
  //    (documentContent is NOT a column on routines — it lives on the
  //    Document entity referenced via routine.documentId).
  //
  //    DocumentsService.update creates a new DocumentVersion (it does NOT
  //    overwrite in place) and updates `documents.current_version_id`.
  //    Signature: update(id, dto: UpdateDocumentDto, identity: DocumentIdentity)
  //    Identity is the acting user — used for permission checks and recorded
  //    as `updatedBy` on the new version row.
  if (dto.documentContent !== undefined) {
    if (!routine.documentId) {
      throw new BadRequestException(
        `Routine ${routineId} has no linked document — cannot update documentContent`,
      );
    }
    await this.documentsService.update(
      routine.documentId,
      { content: dto.documentContent },
      { type: 'user', id: userId },
    );
  }

  // 3. If triggers were provided, replace the entire trigger list for this
  //    routine. Wholesale replacement — the caller is responsible for
  //    passing the full desired list.
  if (dto.triggers !== undefined) {
    await this.routineTriggersService.replaceAllForRoutine(
      routineId,
      dto.triggers,
      tenantId,
    );
  }

  return updated;
}
```

```typescript
async completeCreation(
  routineId: string,
  dto: CompleteCreationDto,
  userId: string,
  tenantId: string,
): Promise<schema.Routine> {
  // Step 1: fetch (throws NotFoundException if missing or wrong tenant)
  const routine = await this.getRoutineOrThrow(routineId, tenantId);

  // Step 2: creator ownership check (throws ForbiddenException)
  this.assertCreatorOwnership(routine, userId);

  // Step 3: idempotent short-circuit
  if (routine.status === 'upcoming') {
    return routine;
  }

  // Step 4: status must be draft
  if (routine.status !== 'draft') {
    throw new BadRequestException(
      `Cannot complete creation: routine is in ${routine.status} status. Only draft routines can be completed.`,
    );
  }

  // Step 5: required-field validation
  const missing: string[] = [];
  if (!routine.title?.trim()) missing.push('title');
  if (!routine.botId) missing.push('botId');

  let docContent: string | null = null;
  if (routine.documentId) {
    const doc = await this.documentsService.getById(routine.documentId);
    docContent = doc?.content ?? null;
  }
  if (!docContent?.trim()) missing.push('documentContent');

  if (missing.length > 0) {
    throw new BadRequestException({
      message: 'Missing required fields',
      errors: missing.map((field) => `${field} is required`),
    });
  }

  // Step 6: transition
  const [updated] = await this.db
    .update(schema.routines)
    .set({ status: 'upcoming', updatedAt: new Date() })
    .where(eq(schema.routines.id, routineId))
    .returning();

  // Step 7: archive creation channel if present (non-fatal on failure)
  if (routine.creationChannelId) {
    try {
      await this.channelsService.archiveCreationChannel(
        routine.creationChannelId,
        tenantId,
      );
    } catch (err) {
      this.logger.warn(
        `Failed to archive creation channel ${routine.creationChannelId} for routine ${routineId}`,
        err,
      );
    }
  }

  // Step 8: delete clone agent if it exists (non-fatal on failure)
  const cloneAgentId = `routine-creation-${routineId}`;
  try {
    await this.clawHiveService.deleteAgent(cloneAgentId);
  } catch (err) {
    this.logger.warn(
      `Failed to delete clone agent ${cloneAgentId} for routine ${routineId}`,
      err,
    );
  }

  // Step 9: audit log
  this.logger.log(
    `Routine ${routineId} creation completed by ${userId}${
      dto.notes ? ` — notes: ${dto.notes}` : ''
    }`,
  );

  return updated;
}
```

```typescript
async createWithCreationTask(
  dto: CreateWithCreationTaskDto,
  userId: string,
  tenantId: string,
): Promise<{
  routineId: string;
  creationChannelId: string;
  creationSessionId: string;
}> {
  // 1. Validate source bot exists and belongs to this tenant
  const sourceBot = await this.botsService.getBotById(dto.agentId);
  if (!sourceBot) {
    throw new NotFoundException(`Agent ${dto.agentId} not found`);
  }
  await this.assertBotInTenant(dto.agentId, tenantId);

  // 2. Fetch source agent definition from claw-hive (for blueprint + configs
  //    + model that the clone should inherit)
  const sourceAgentId = sourceBot.managedMeta?.agentId;
  if (!sourceAgentId) {
    throw new BadRequestException(
      'Selected agent is not claw-hive-managed and cannot be cloned',
    );
  }
  const sourceAgent = await this.clawHiveService.getAgent(sourceAgentId);
  if (!sourceAgent) {
    throw new NotFoundException(
      `Claw-hive agent ${sourceAgentId} not found`,
    );
  }

  // 3. Auto-generate title based on tenant routine count
  const [{ count: existingCount }] = await this.db
    .select({ count: sql<number>`count(*)::int` })
    .from(schema.routines)
    .where(eq(schema.routines.tenantId, tenantId));
  const title = `Routine #${existingCount + 1}`;

  // 4. Create the draft routine (reuses RoutinesService.create's draft path
  //    from Task 2 — it skips trigger registration automatically)
  const draft = await this.create(
    {
      title,
      botId: dto.agentId,
      status: 'draft',
    },
    userId,
    tenantId,
  );

  const cloneAgentId = `routine-creation-${draft.id}`;
  let cloneRegistered = false;
  let channelId: string | null = null;

  try {
    // 5. Create or reuse DM channel between user and bot shadow user
    const channel = await this.channelsService.createDirectChannel(
      userId,
      sourceBot.userId,
      tenantId,
    );
    channelId = channel.id;

    // 6. Build session id deterministically so the gateway-side message
    //    forwarder (post-broadcast.service.ts) will route messages in
    //    this DM into this exact session.
    const sessionId = `team9/${tenantId}/${cloneAgentId}/dm/${channel.id}`;

    // 7. Register the clone agent. The clone inherits the source agent's
    //    blueprint, component configs (persona/model/SOUL/etc), and model.
    //    We overlay a team9-routine-creation config so the clone knows
    //    which routine it is bound to.
    await this.clawHiveService.registerAgent({
      id: cloneAgentId,
      name: `Routine Creation for ${sourceBot.displayName ?? 'Staff'}`,
      blueprintId: sourceAgent.blueprintId,
      tenantId,
      model: sourceAgent.model,
      componentConfigs: {
        ...sourceAgent.componentConfigs,
        'team9-routine-creation': {
          routineId: draft.id,
          isCreationChannel: true,
        },
      },
    });
    cloneRegistered = true;

    // 8. Persist creation metadata on the draft routine
    await this.db
      .update(schema.routines)
      .set({
        creationChannelId: channel.id,
        creationSessionId: sessionId,
        updatedAt: new Date(),
      })
      .where(eq(schema.routines.id, draft.id));

    // 9. Send kickoff event so the clone emits its greeting message
    await this.clawHiveService.sendInput(
      sessionId,
      {
        type: 'team9:routine-creation.start',
        source: 'team9',
        timestamp: new Date().toISOString(),
        payload: {
          routineId: draft.id,
          creatorUserId: userId,
          creationChannelId: channel.id,
        },
      },
      tenantId,
    );

    return {
      routineId: draft.id,
      creationChannelId: channel.id,
      creationSessionId: sessionId,
    };
  } catch (err) {
    // Rollback path
    this.logger.error(
      `createWithCreationTask failed for draft ${draft.id}, rolling back: ${
        err instanceof Error ? err.message : String(err)
      }`,
    );

    // Delete clone if it was registered
    if (cloneRegistered) {
      await this.clawHiveService
        .deleteAgent(cloneAgentId)
        .catch((e) =>
          this.logger.warn(
            `Failed to delete clone ${cloneAgentId} during rollback`,
            e,
          ),
        );
    }

    // Delete draft routine row
    await this.db
      .delete(schema.routines)
      .where(eq(schema.routines.id, draft.id));

    // Channel is intentionally left in place — it's idempotent and a retry
    // will reuse it.

    throw err;
  }
}

private async assertBotInTenant(
  botId: string,
  tenantId: string,
): Promise<void> {
  const [row] = await this.db
    .select({
      tenantId: schema.installedApplications.tenantId,
    })
    .from(schema.bots)
    .innerJoin(
      schema.installedApplications,
      eq(schema.bots.installedApplicationId, schema.installedApplications.id),
    )
    .where(eq(schema.bots.id, botId))
    .limit(1);

  if (!row || row.tenantId !== tenantId) {
    throw new BadRequestException(
      `Bot ${botId} does not belong to the current tenant`,
    );
  }
}
```

```typescript
  BeforePromptResult,
  ComponentContext,
  ComponentData,
  ConversationEntry,
  InputEvent,
  UserEntry,
} from "@team9claw/types";
import type {
  HiveInputEvent,
  Team9RoutineCreationComponentConfig,
} from "@team9claw/claw-hive-types";
import type { Team9DependencyApi } from "../team9/dependency.js";

/**
 * Mutable component state for team9-routine-creation.
 *
 * Populated from either the static config (Routine UI path: routineId in
 * extraComponentConfigs at registration time) or from the bootstrap event
 * payload (defensive fallback).
 */
export interface Team9RoutineCreationComponentData extends ComponentData {
  /** The draft routine this session is refining, if any. */
  routineId?: string;
  /** The user who kicked off the creation — used for ownership validation. */
  creatorUserId?: string;
  /** The creation channel ID — used for audit logging. */
  creationChannelId?: string;
}

export class Team9RoutineCreationComponent extends BaseComponent<
  Team9RoutineCreationComponentConfig,
  Team9RoutineCreationComponentData
> {
  readonly dependencies = ["team9"] as const;
  readonly subscribedEvents = ["team9:routine-creation.start"];

  constructor(config: Team9RoutineCreationComponentConfig = {}, id?: string) {
    super(
      {
        typeKey: "team9-routine-creation",
        name: "Team9 Routine Creation",
        priority: 92,
      },
      config,
      id,
    );
  }

  // ── Mode detection ──────────────────────────────────────────────

  /**
   * Effective routineId: prefer static config, fall back to runtime data
   * (set by onEvent from the kickoff event payload).
   */
  private getEffectiveRoutineId(
    ctx: ComponentContext<
      Team9RoutineCreationComponentConfig,
      Team9RoutineCreationComponentData
    >,
  ): string | undefined {
    return ctx.config.routineId ?? ctx.data.routineId;
  }

  private isRoutineUiMode(
    ctx: ComponentContext<
      Team9RoutineCreationComponentConfig,
      Team9RoutineCreationComponentData
    >,
  ): boolean {
    return Boolean(this.getEffectiveRoutineId(ctx));
  }

  // ── Event handling ──────────────────────────────────────────────

  async onEvent(
    ctx: ComponentContext<
      Team9RoutineCreationComponentConfig,
      Team9RoutineCreationComponentData
    >,
    event: InputEvent,
  ): Promise<void> {
    const hiveEvent = event as HiveInputEvent;
    if (hiveEvent.type !== "team9:routine-creation.start") return;

    const payload = hiveEvent.payload as Record<string, unknown> | undefined;
    if (!payload || typeof payload !== "object") return;

    await ctx.updateData({
      routineId:
        typeof payload.routineId === "string" ? payload.routineId : undefined,
      creatorUserId:
        typeof payload.creatorUserId === "string"
          ? payload.creatorUserId
          : undefined,
      creationChannelId:
        typeof payload.creationChannelId === "string"
          ? payload.creationChannelId
          : undefined,
    });
  }

  formatEventEntry(
    _ctx: ComponentContext<
      Team9RoutineCreationComponentConfig,
      Team9RoutineCreationComponentData
    >,
    event: InputEvent,
  ): ConversationEntry | null {
    const hiveEvent = event as HiveInputEvent;
    if (hiveEvent.type !== "team9:routine-creation.start") return null;

    const entry: UserEntry = {
      id: `evt_${crypto.randomUUID()}`,
      type: "user",
      timestamp: Date.now(),
      content:
        "[SYSTEM] Start routine creation conversation. Greet the user warmly, ask what routine they want to set up, and guide them through title, document content (instructions), and triggers. Call getRoutine first to see the current draft state before proposing changes.",
    };
    return entry;
  }

  // ── System prompt injection ────────────────────────────────────

  override onBeforePrompt(
    ctx: ComponentContext<
      Team9RoutineCreationComponentConfig,
      Team9RoutineCreationComponentData
    >,
  ): BeforePromptResult {
    // Sanity check: make sure team9 dependency is available. If not, we
    // can't actually call any tools, so skip the prompt injection.
    const team9 = ctx.getDependency<Team9DependencyApi>("team9");
    if (!team9?.getAuthToken() || !team9.getBaseUrl()) {
      return {
        contextInjection: {
          target: "system",
          content:
            "[routine-creation] Team9 connectivity is missing — routine creation tools are unavailable.",
        },
      };
    }

    if (this.isRoutineUiMode(ctx)) {
      const routineId = this.getEffectiveRoutineId(ctx)!;
      return {
        contextInjection: {
          target: "system",
          content: [
            "<routine_creation_mode>refinement</routine_creation_mode>",
            `You are refining an existing draft routine (routineId=${routineId}).`,
            "",
            "Workflow:",
            "1. ALWAYS call `getRoutine` first at the start of each turn to see the current state.",
            "2. Ask clarifying questions to fill in missing fields: title, description, documentContent (the actual instructions the routine will execute), triggers.",
            "3. Call `updateRoutine` with your proposed changes. Partial updates are fine — only include fields you want to change.",
            "4. After each update, render the plan to the user as readable narrative prose (not JSON), showing: Routine Name, What it does, When it runs (triggers), and Key steps.",
            "5. When the user confirms the plan looks good, tell them to click **Complete Creation** in the Routine UI to finalize. Do NOT attempt to complete creation yourself — that's a user action.",
            "",
            "Rules:",
            "- This clone agent exists only for this routine. It will be deleted when the routine is completed.",
            "- You do NOT have `createRoutine` in this mode — the routine already exists.",
            "- Do NOT proactively create other routines or edit unrelated entities.",
          ].join("\n"),
        },
      };
    }

    // DM mode
    return {
      contextInjection: {
        target: "system",
        content: [
          "<routine_creation_mode>dm</routine_creation_mode>",
          "The user may ask you to create a routine during normal DM conversation.",
          "",
          "When the user asks, gather:",
          "- `title` (required, short human-readable name)",
          "- `documentContent` (required, the actual instructions the routine will execute when it runs — be clear and complete)",
          "- `triggers` (optional, defaults to `[{type: 'manual'}]` if user doesn't specify)",
          "- `botId` (optional, defaults to yourself — you are the executing agent unless the user names someone else)",

          "If you don't have enough information to create a runnable routine, ask 1–2 clarifying questions first, then call the tool. Do NOT create empty or placeholder routines.",
```

```typescript
// Inside class:

private cachedTools?: AgentTool[];

override getTools(
  ctx: ComponentContext<
    Team9RoutineCreationComponentConfig,
    Team9RoutineCreationComponentData
  >,
): AgentTool[] {
  const team9 = ctx.getDependency<Team9DependencyApi>("team9");
  if (!team9?.getAuthToken() || !team9.getBaseUrl()) {
    return []; // no tools if team9 is not wired up
  }

  if (!this.cachedTools) {
    const apiClient = new Team9ApiClient(
      team9.getBaseUrl()!,
      team9.getAuthToken()!,
    );

    const getRoutineId = () => this.getEffectiveRoutineId(ctx);

    if (this.isRoutineUiMode(ctx)) {
      this.cachedTools = [
        createGetRoutineTool({ apiClient, getRoutineId }),
        createUpdateRoutineTool({ apiClient, getRoutineId }),
      ];
    } else {
      this.cachedTools = [
        createCreateRoutineTool({ apiClient, team9 }),
      ];
    }
  }

  return this.cachedTools;
}
```

**⚠ CAUTION:** Caching tools in an instance field is tricky because `ctx` changes between calls. The closure captured in the tool factory (`getRoutineId = () => this.getEffectiveRoutineId(ctx)`) will capture the `ctx` from the first `getTools` call. If `ctx` is the same across calls within one session (which it should be in the worker's session loop), this is fine. If not, we'd need to rebuild tools per-call. Verify empirically in tests: if two sequential calls to `getTools` with different ctx values produce tools that operate on the wrong routineId, drop the cache.

**Safer alternative:** rebuild the tools on every `getTools` call (no cache):

```typescript
override getTools(ctx): AgentTool[] {
  const team9 = ctx.getDependency<Team9DependencyApi>("team9");
  if (!team9?.getAuthToken() || !team9.getBaseUrl()) return [];

  if (this.isRoutineUiMode(ctx)) {
    return [
      createGetRoutineTool({ apiClient, getRoutineId }),
      createUpdateRoutineTool({ apiClient, getRoutineId }),
    ];
  }
  return [createCreateRoutineTool({ apiClient, team9 })];
}
```

```typescript
    },
```

```typescript
            ) : bots.length === 0 ? (
              <div className="text-sm text-muted-foreground">
                {t("agentic.noAgentsAvailable")}
              </div>
            ) : (
              <Select
                value={defaultBotId}
                onValueChange={setSelectedBotId}
              >
                <SelectTrigger className="w-full">
                  <SelectValue />
                </SelectTrigger>
                <SelectContent>
                  {bots.map((bot) => (
                    <SelectItem key={bot.botId} value={bot.botId}>
                      {bot.displayName || bot.username}
                    </SelectItem>
                  ))}
                </SelectContent>
              </Select>
            )}
          </div>
        </div>

        <DialogFooter>
          <Button variant="ghost" onClick={onClose}>
            {t("agentic.cancel")}
```

```tsx
import { useTranslation } from "react-i18next";
import { useNavigate } from "@tanstack/react-router";
import { useMutation, useQueryClient } from "@tanstack/react-query";
import { Card } from "@/components/ui/card";
import { Button } from "@/components/ui/button";
import { Badge } from "@/components/ui/badge";
import {
  Tooltip,
  TooltipContent,
  TooltipProvider,
  TooltipTrigger,
} from "@/components/ui/tooltip";
import { Trash2, MessageSquare } from "lucide-react";
import type { Routine } from "@/types/routine";
import { api } from "@/services/api";
import { toast } from "sonner";

interface DraftRoutineCardProps {
  routine: Routine;
}

export function DraftRoutineCard({ routine }: DraftRoutineCardProps) {
  const { t } = useTranslation("routines");
  const navigate = useNavigate();
  const queryClient = useQueryClient();

  const deleteMutation = useMutation({
    mutationFn: () => api.routines.delete(routine.id),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ["routines"] });
    },
    onError: (err) => {
      toast.error(err instanceof Error ? err.message : String(err));
    },
  });

  const handleCompleteCreation = () => {
    if (!routine.creationChannelId) return;
    void navigate({
      to: "/messages/$channelId",
      params: { channelId: routine.creationChannelId },
    });
  };

  const handleDelete = () => {
    if (!window.confirm(t("draft.deleteConfirm"))) return;
    deleteMutation.mutate();
  };

  return (
    <Card className="p-4">
      <div className="flex items-center justify-between gap-3">
        <div className="min-w-0 flex-1">
          <div className="flex items-center gap-2">
            <h3 className="truncate text-base font-medium">{routine.title}</h3>
            <Badge variant="outline" className="uppercase">
              {t("draft.badge")}
            </Badge>
          </div>
          {routine.description && (
            <p className="mt-1 truncate text-xs text-muted-foreground">
              {routine.description}
            </p>
          )}
        </div>
        <div className="flex shrink-0 items-center gap-2">
          {routine.creationChannelId ? (
            <Button
              size="sm"
              onClick={handleCompleteCreation}
              aria-label={t("draft.completeCreation")}
            >
              <MessageSquare size={14} className="mr-1" />
              {t("draft.completeCreation")}
            </Button>
          ) : (
            <TooltipProvider>
              <Tooltip>
                <TooltipTrigger asChild>
                  <Button size="sm" disabled>
                    <MessageSquare size={14} className="mr-1" />
                    {t("draft.completeCreation")}
                  </Button>
                </TooltipTrigger>
                <TooltipContent>{t("draft.noChannelTooltip")}</TooltipContent>
              </Tooltip>
            </TooltipProvider>
          )}
          <Button
            size="sm"
            variant="ghost"
            onClick={handleDelete}
            disabled={deleteMutation.isPending}
            aria-label={t("draft.delete")}
          >
            <Trash2 size={14} />
          </Button>
        </div>
      </div>
    </Card>
```

```typescript
private async provisionRoutines(
  workspaceId: string,
  userId: string,
  onboardingRecordId: string,
  stepData: WorkspaceOnboardingStepData,
): Promise<void> {
  const tasks = stepData.tasks;
  if (!tasks) return;

  // Collect intended tasks
  const selectedGenerated =
    tasks.generatedTasks?.filter((t) =>
      tasks.selectedTaskIds?.includes(t.id),
    ) ?? [];

  interface IntendedTask {
    sourceChildId: string;
    title: string;
  }
  const intended: IntendedTask[] = [
    ...selectedGenerated.map((t) => ({
      sourceChildId: t.id,
      title: t.title.trim(),
    })),
    ...(tasks.customTask?.trim()
      ? [{ sourceChildId: 'custom', title: tasks.customTask.trim() }]
      : []),
  ];
  if (intended.length === 0) return;

  // Resolve user's personal-staff bot
  const personalStaffApp =
    await this.installedApplicationsService.findByApplicationId(
      workspaceId,
      'personal-staff',
    );
  if (!personalStaffApp) {
    this.logger.warn(
      `provisionRoutines: no personal-staff app for workspace ${workspaceId}, skipping`,
    );
    return;
  }
  const personalBot = await this.personalStaffService.findPersonalStaffBot(
    userId,
    personalStaffApp.id,
  );
  if (!personalBot) {
    this.logger.warn(
      `provisionRoutines: user ${userId} has no personal-staff bot, skipping`,
    );
    return;
  }

  // Idempotently create drafts
  for (const task of intended) {
    const sourceRef = `onboarding:${onboardingRecordId}:${task.sourceChildId}`;

    const [existing] = await this.db
      .select({ id: schema.routines.id })
      .from(schema.routines)
      .where(
        and(
          eq(schema.routines.tenantId, workspaceId),
          eq(schema.routines.sourceRef, sourceRef),
        ),
      )
      .limit(1);

    if (existing) {
      this.logger.debug(
        `provisionRoutines: routine with sourceRef ${sourceRef} already exists, skipping`,
      );
      continue;
    }

    await this.routinesService.create(
      {
        title: task.title,
        botId: personalBot.botId,
        status: 'draft',
      },
      userId,
      workspaceId,
      { sourceRef },
    );
  }

  this.logger.log(
    `provisionRoutines: created ${intended.length} draft routine(s) for workspace ${workspaceId}`,
  );
}
```
